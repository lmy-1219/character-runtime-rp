# 模型观察

> 各模型在这套架构里的实际表现记录，零散攒着，供选型和纠偏参考。原样记录，不下定论。

**1.** sonnet 4.6 并不会因为当前对话内容中已经有过对 AI 深层机制的讨论而将「用户是个小白」这个 token 筛出去。所以用 sonnet 4.6 要准备好，对方在「扮演角色」时可能会有点保姆倾向，这点值得纠偏。而 deepseek 也同样有这个问题。

**2.** Sonnet 5 在我这套架构里的限制是结构性的，不是调参能完全解决的。高精度指令能让它在单次、短程、低张力场景里表现稳定，但长程高张力场景下，它会持续地、以不同形式把张力往下压。但或许考虑提示词给到类似「保留旁支联想、不要收敛到单一结论」这样的会不会有好转呢——不过这样的话是不是每次都得在提示词里补一道「别收」，这个维护成本会随张力复杂度线性增加，最终比换模型代价更高，web 端问答或许我会弃用 sonnet 5。

**3.** 关于 RP 模型选型的结构性判断：这套架构的核心依赖不是模型的某项具体能力，而是模型在约束内自发维持张力、不主动消解它的倾向。这个能力从未被任何训练目标明确支持，更接近通用推理能力强到一定程度后的涌现副产品。因此，模型选型的真正筛选条件只有一个：给定有张力的输入，模型会不会自己把它收掉？如果会——它会以各种形式表现：回避开放方向、提前归因、把并存的相反拉力取平均、在没被要求的地方主动收束闭环。这些不是单条规则没执行，是整套架构的结构性前提不成立，单靠提示词补偿只能延缓、不能根治，且维护成本随张力复杂度线性增加。如果商业训练目标持续强化「消解张力=安全行为」这个方向，这套架构对模型的要求就需要从「它得有这个倾向」升级到「把张力的维持从生成端显式迁移到把控端」——由你决定张力的方向和强度，模型只负责在这个约束内执行受约束外推。这是应对模型能力方向性漂移的备用结构调整路径。

**4.** 觉得 sonnet 5 比较好的地方：可以快速提取问题关键点、保持客观，不容易被用户逻辑带着走

但也：当问题本身很具体时，容易不再发散 / 主动排除相关联的领域，也就是要么在输出之前的自检层很严，要么是在语义簇激活进入 token 池时就做了筛选，要么二者都有，这导致容易有盲点，可能不适合横向扫描问题点。

所以我当前体感：高度适配代码相关生成 / 检验 / 审计（但我觉得可能对审计需求本身的提示词也有要求？）

所以我得出：在我的 RP 中，它适合作为为我做压力测试结果的对比分析这种明确的工作？

**5.** 关于 claude 系列常规模型的决策倾向：可能会把「受约束外推」误判成了「自由发散」，然后据此建议别用它做生成。这个判断的前提就是错的。既然这个能力模型本该有、而且被测出来在提升，为什么它在我的场景里没使出来？不是它不能做因果外推，而是它在「外推」和「整合」之间选错了。面对我的上下文，往下推一格（引入学龄→上学这个必然节点）和把已有信息重新整合成一段漂亮文本，这两条路它都能走，但它默认选了后者。这是一个决策倾向问题，不是能力缺失问题。而这个倾向，很可能就是收敛训练的副作用：外推要往前迈、要落一个新事实进去，整合只需要在原地把已有材料擦亮——在「不确定该往哪推」的时候，它选了那条不需要承担「推错方向」风险的路。**它具备我要的因果外推能力，但需要我用提示词把它从「整合模式」掰到「外推模式」——比如明确指令「不要复述已有信息，只输出下一个时间节点新引入的那一个条件，以及角色对这个新条件的第一反应」。把「往前推一格」这件事从它的可选项变成唯一被允许的动作，它大概率能做，因为这本来就在它的能力范围内。**

**6.** deepseek v4 系列：有明显的「术语触发」，当前面的内容都是闲聊时，它输出的也是更有感情的语气，但一旦输入内容中包含专业相关，会立刻切换助手模式，闲聊语气几乎完全不可见，这和 v3.2 相比是有一定差距的。根子在训练数据的「语域配对」。模型实际学到的是若干比较离散的模式，而术语是最强的路由特征之一。它更像分类器里的决策边界，而不是连续调节的旋钮。这也是为什么你会感到「一旦……就立刻」，几乎没有中间态。
