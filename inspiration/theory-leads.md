# 理论线索

> 尚未验证、尚未进入架构的跨领域理论方向。原样记录，不下结论。

## 一、什么东西有资格「被系统相信、活到下一轮」

Role-Playing Language Agents、LLM Interactive Drama / Narrative Agent、Believable Agents、LLM memory 这些，对应我现有架构都有部分重合，但未外化意图不持久化；连续性必须由戏里发生过的证据承载——什么东西有资格被系统相信，并活到下一轮，这一点我未找到相关定义和文献。

文学理论里的「什么在虚构世界里为真」（fictional truth，Kendall Walton 的 make-believe 理论），以及 Truth Maintenance System (TMS) / Belief Revision / AGM framework（Alchourrón-Gärdenfors-Makinson，信念修正的奠基理论）、数据库和知识图谱领域的一整套 data provenance 理论、检索增强生成（RAG）和对话系统里的 groundedness / attribution / faithfulness 研究。

用虚构真值理论定义「什么算发生过」，用信念修正理论定义「它如何存活到下一轮」——是否能从这里面有所借鉴？或许能根据这些再进一步对我的提示词工程作用于角色真实性上进行进一步优化？

## 二、认知分层与表演嵌套

核心问题：让模型实现「在戏中戏里演一个蹩脚演员」——即 L1 角色（虚构演员）在 L2（戏中戏）里产生可归因的、指向 L1 内心状态的受控瑕疵。

关键推论链：认知隔离（零泄漏）→ 受控泄漏（精确宽度的缝）→ 嵌套扮演（泄漏方向反转）→ 高质量地生成低质量（负向的精确）

模型在这条链上的能力边界大致在第三层和第四层之间——它能模仿蹩脚的样子，但它演的是「蹩脚」这个概念，不是一个具体的人在具体处境下具体地演不好。原因是它没有「想演好但演不好」的身体记忆，只有训练分布里关于「生硬 / 出戏」的文本模式。

对架构设计的实用结论：

这套嵌套理解作为分析工具有价值（帮助看清泄漏的方向性和强度可调性），但不适合作为生产架构——层数是纯成本，稳定性随嵌套层数快速衰减。

真正干净的设计仍然是：在信息入口处隔离认知，不给终点，不给超出当事人视角的材料。不是让模型别做什么，而是让它根本没有材料去做。没有粉红大象，不需要任何阀门。
