# 第一部分 · 先搞明白 AI 是怎么「想」的

AI 生成的本质，不是「思考」，是「概率预测」。

每一轮输出，大模型干的事就一件：看完前面所有文字（上下文），算一个概率分布，从概率最高的那批词里挑一个当下一个 token。它没有意识、没有意图、没有灵光一现，也没有玄学。它只有一个能力——根据已有文本，猜「最可能接在后面的是什么」。

所以我要做的，根本不是「教 AI 理解角色」，而是：**通过提示词，重新塑造模型「看到的」那片上下文空间。** 这片空间里放什么事实、什么规则、什么边界，直接决定了它预测下一个 token 时，候选词被缩到哪个范围。

可以把它想成**一张概率地图**。
没有约束的时候，AI 的「下一个词」候选范围是整个语料库——什么都能接，完全发散。
一张好的角色卡，本质上就是在这张地图上画圈：把「这个角色不可能说的话」圈出去，把「他最可能说的话」权重放大。圈画完，圈里的路怎么走、走哪条岔道，交给模型自己算。

**我不替他选词，我只决定他的选项池长什么样。**

这套系统迭代到现在我最满意的一点，就是用多层约束把这个概率空间缩得**足够窄、但没缩死**——窄到角色不跑偏，宽到他还有自由度、有意外、有他自己。

用更硬的话说一遍同一件事：模型每一步在做的是 `P(next_token | context)` 的条件概率最大化。我做的全部工作，是往 context 里堆足够稠密的约束，把概率质量从「不属于这个角色的词」上移走、压到「属于他的词」上，但**不指定具体路径**——即不把角色写成一组静态标签（近似 one-hot），而是写成一台带约束条件的状态机：同样的输入，状态不同，输出分布就不同。后面三根支柱，都是在往 context 里加不同种类的约束权重。

### 为什么「塞得越满」反而越糟

这是新手最常踩的坑，也最能说明「拆底层」和「堵漏」的区别。

**表面现象**：卡写得很用心、信息塞得满满当当，结果模型表现偏「平均」——要么哪个特点都不突出、整个人很平庸，要么某个特点被无限放大、角色直接崩成一个标签人。

**大多数人的反应**：觉得是「写得还不够细」，于是继续加——再补几条性格、再写几段背景、再加几条禁止项。越补越满，越满越糊。

**往下挖一层**：回到上面那张概率地图。你画的每一个圈，都是在抢模型的注意力权重；而注意力是**有限的、会被稀释的**。当你把什么都标成「重点」，等于一个圈都没画——模型分不清哪条约束该重、哪条该轻，只好雨露均沾，于是给你一个各项拉平的「平均脸」。信息密度过高的卡，崩的不是细节，是**优先级**。

**从成因下手的解法**：不是「再加」，是**「敢减、敢压」**。我自己砍东西时，大致按这几条过一遍——这条信息，模型不知道会不会犯常识错误？大概率不会的，往往就不必常驻。这条算角色的「底色」还是「细节」？底色我给高权重、常驻；细节降权，或者交给触发机制按需浮出（这就是后面[第四部分](05-layered-memory.md)世界书分层在干的事）。这两条特征，是不是在抢同一份注意力、互相打架？是的话，我会明确谁主谁次，而不是留给模型猜。

说人话就是：**写卡的功夫，一半在「写什么」，另一半——往往更难的那一半——在「敢删什么、敢把什么调暗」。** 把概率质量集中到真正定义这个角色的那几样东西上，剩下的让它退到背景里去。

---

那具体靠什么「画圈」？三根支柱，接下来一根一根拆：

- **底层驱动力**——决定「他为什么会动」（[第二部分](03-causality.md)）
- **涟漪机制**——决定「什么该被放大、惦记多久」（[第三部分](04-ripple.md)）
- **分层记忆**——决定「他怎么在百万字里还记得住事」（[第四部分](05-layered-memory.md)）
