# 第六部分 · 模型纠偏：同一套卡，不同模型不同打法

> 同一套提示词，不同模型跑出来天差地别。纠偏的本质是：针对每个模型已知的「偏」，反向补偿——天生收敛的给它松绑，天生发散的给它套绳。

各家训练数据、对齐方式、RLHF 策略都不一样：有的天生收敛、有的天生发散；有的吃禁止项、有的完全不吃。所以同一张卡，得给不同模型配不同纠偏提示词，专补这个模型的脾气缺陷。比如 Claude 被训得「稳健无害」，演强尼这种攻击性角色会主动往「温和／安全」偏；Gemini 被训得「有创意多样」，会往「戏剧化／夸张」偏。

我实测在用的三个（版本是我当时测的，不一定是最新）：

**Gemini 3.1 Pro Preview —— 日常轻松氛围**
强：通用知识丰富，能主动调用夜之城概念做修辞，AU 沉浸感强；低推理强度场景里，它那点「不稳定」反而让角色更鲜活。弱：会抓住「最能制造戏剧性」的那个特征一路放大、不收敛；不吃禁止项、只吃正面描述——你写「禁止 XX」它当没看见，写「应该 XX」它才听。纠偏方向：每轮重读场景、禁止惯性滑行、禁止把某个特征一路升级。

### 关于禁止项：轻易别穷举禁止词

所有角色卡和系统提示词，正常情况下每一轮都会全量注入上下文。所以你列的每一个"禁止出现 XX"，模型每轮都会重新读到一遍 XX——有些词本来不在你框好的 token 池里，你这一列，反而把它请进了模型的预测范围。一张长长的禁止清单，对模型来说更像一张"每轮复习这些词"的清单。

而且否定句对模型是有副作用的：模型要处理"不要 XX"，得先把 XX 这个概念点亮，才谈得上抑制它；至于那个"不要"压不压得住，要看模型的反向指令跟随能力。对 Gemini 这类反向跟随弱的模型，只给禁止、不给出口，禁令基本失效（我没法主观断定这纯粹是枚举所致、还是这类模型本身反向跟随弱，但实测结论就是失效）。

并且，单纯禁止某个词，对有反向跟随能力、但推理能力偏弱的模型，能禁掉的可能只有你点名的那个词，而不是整个语义簇。比如你写"禁止使用'狠狠'"，它确实不写"狠狠"了，但转头给你输出"极其用力"——你想掐的是"用副词硬撑力度"这一整类毛病，它只听懂了字面那两个字。所以禁止句越是落在具体词上，越容易出现这种"按字面绕过"；要掐的是一类写法时，得把那条规则本身描述出来（见[第五部分](06-character-card.md)文风闸：禁的不是某个副词，是"力度靠副词撑"这件事），而不是去枚举词。

所以判断一个东西该不该用禁止句，看一条：它被激活之后，会不会把你不想要的输出词的概率也抬上去。

1. "不要写冲突"——冲突是叙事内容词，点亮它，"争吵、对抗、摔门"这一簇词的权重跟着涨，而这些词本来就会跑进正文。所以你提了它，它反而更容易出现。这种要改成正向描述。
2. "不要惯性滑行"——惯性滑行描述的是模型自己的生成习惯，不是场景里会发生的事，点亮它不会有任何叙事词跟着涨，它不会变成"Johnny 惯性地说……"。这种禁止句没有副作用，写了没问题。

所以不是"规则层能禁、渲染层不能禁"，而是：禁一个东西，先看它激活后会不会拉高你不想要的词的概率。会的，改成正向描述给出口；不会的，禁止句是干净的，随便写。

我现在写禁止项用的是三段式：**反向约束 + 正向描述 + 总结**。反向约束划掉不要的方向，正向描述给概率质量一个合法去处（靠抬高想要的词、挤占掉不想要的词，而不是硬摁），总结收口。（正向描述具体怎么写，见[第五部分](06-character-card.md)文风闸那条，不赘述。）

**DeepSeek V4 Flash —— 日常温馨氛围**
强：快；有较强的 helpful 反射（演攻击性角色时不会主动在台词里制造冲突，这本是劣势，对温馨场景反成优势）；输出稳，吃得住文风约束。弱：通用知识弱、幻觉强。纠偏：用得不多，跟 Claude 共用主提示词效果就挺好。

**Claude Opus 4.6 —— 高压／高张力／高信息密度（战术规划、多角色、被击中反应）**
强：推理极强，能演出复杂性和立体度，兼顾战术逻辑；天然文风硬朗清爽，文风约束对它基本是多余项。弱：太吃禁止项、会过度收敛、氛围易发沉；缺自主性、爱「响应」不爱「主动」；过于稳健成熟、太会收束闭环；贵。纠偏方向（跟 DeepSeek 共用）：它的毛病是「收着」——检测到场景情绪后，给一个恰当、精准、善解人意的回复，再把场面收拾干净。所以得反着推：不准给完美匹配的回复、不准把收尾收干净、不准温柔接住 V 的脆弱时刻。它要是检测到「她在脆弱」、然后产出一个温柔体贴的回应，那本身就是失败。

另外几个三家共用的纠偏模块：

- **情感高峰不许劫持读取**：用户在长段描写里埋了一句话（「我其实有点怕」），模型爱跳过前面的动作直接抓这句做情感反应；要求读完整段，可以回应前面的动作、或者干脆不接这茬。
- **说话是情绪驱动、不是助手结构化**：禁「问题分析-解决方案」那套结构、禁总结用户情绪；回复可以中断、偏题、话说一半断掉，但回合控制权要还给用户（不替用户收场、送人睡觉、跳时间）。
- **他只知道自己能知道的事**：禁全知视角，超出认知范围只能问、观察、猜。
- **主动性 vs 控制权**：禁止替用户做决定／同意／感受，但不禁止强尼发起动作（主动走向门、打电话、拒委托、提路线、靠近、先吻、说他要去哪）。他发起，用户回应。
