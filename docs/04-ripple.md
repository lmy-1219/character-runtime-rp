# 第三部分 · 支柱二：涟漪机制

> 发生过的事不该过几轮就沉底。涟漪机制给上下文叠一个随轮数衰减的权重场——近期没解决的事，势能高、持续影响他；老的、了结的，淡出。

声明在前：这一块在我自己的 RP 里提升体感不大（原因后面说），我只轻量验证过涟漪池本身，没做长程压测。下面是原理 + 轻量落地，放出来主要给角色卡偏薄的人参考。

我碰到的问题：发生过的事，过几轮就沉下去了。10 轮前刚吵过一架，10 轮后他跟我聊音乐，跟没事人一样。模型的注意力被新输入占满，旧事掉出有效上下文，就等于没发生过。

涟漪机制的核心想法：每件真正发生的事，不该只是「一条事实记录」，而该像往水里扔石头——荡开、持续影响后面的行为，直到涟漪自己褪掉。落地分三层：

- **事实层**：插件把「发生了什么」抽成条目，落进世界书（具体在[第七部分](08-plugin-workflow.md)讲）。这层只记客观事实。
- **心理层（涟漪池）**：记这件事在角色心里勾起了什么——他在意什么、回避什么、被什么勾住、什么在他脑子里翻来覆去。注意：这是推断，不是事实，所以它跟事实层物理隔离（为什么隔离，[第八部分](09-character-agency.md)细说，那是整套系统不自我污染的命门）。
- **热度机制**：刚发生 = 热，高频影响行为；时间一长、或事儿被处理掉 = 降温、淡出。要是发生了好多轮 user 还一直装看不见 → 他该生气了。

这里其实藏着一个选择。表面问题是「模型健忘」，看起来加一句「记住之前发生的事」就行。往下挖一层：模型不是「忘了」，是旧事的 token 在新上下文里权重被稀释到接近零——它「看得见」但「不在意」。所以解法不是提醒它记，是给关键的旧事人工续上一个不随轮数自然衰减到零的权重，让它在该响的时候一直响。这就是涟漪池在干的：把「该惦记多久」从模型的随机注意力里拿出来，变成一条可控、显式的衰减曲线。

<details>
<summary>精确点说（给想抠机制的人）</summary>

本质是给上下文叠了一个随轮数衰减的权重场：近期的、没闭环的，势能高；老的、闭环的，势能低。模型每轮不只看当前输入，还看着这片带权重的事实场。我的涟漪池条目带一个 `[热度]` 字段（冷／温／热／沸／睡眠），由一个专门的审稿插件（wrecRipple）按局面调，而不是按固定轮数——「禁止任何每 X 轮必须」是整套自动化的一条通则，[第八部分](09-character-agency.md)会再提。

</details>

老实说，涟漪机制在我自己这儿提升体感不大。因为我的角色卡本身够细，加上常开的总结插件已经把没闭环的事持续往上下文里送了——这两样在我这儿已经把「旧事不沉底」覆盖了大半，它对我是锦上添花。但对角色卡偏薄、总结提示词不够具体的配置，这东西可能是雪中送炭。各自判断。
