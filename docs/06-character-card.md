# 第五部分 · 角色卡怎么写

> 角色卡不是人设档案，是运行规则。把「他是个什么人」翻译成模型能「算」的约束条件——偏好、触发、行为倾向、能力边界——然后让它自己推每一拍。

它最早长这样：「强尼·银手，暴躁、叛逆、嘴硬心软、喜欢弹吉他、讨厌公司。」这种卡 AI 看了等于没看：它认得「暴躁」，但不知道暴躁什么时候出来、什么时候不出来、暴躁完怎么收场。结果是让他永远暴躁，某些模型更狠，一上来就疯狂攻击 user 人格。

### 一、先把角色拆明白：从源材料逆向，别凭感觉写

这张卡的料是三条线：

1. **游戏本体**：从 2077 里抠了 200+ 条强尼第一人称口吻的任务文案，每条做剧情补充 + 主观分析；所有语音听完，分析口吻、语气、不同态度。
2. **桌游前史**：从赛博朋克 2020 的剧情里找到强尼的完整生平。
3. **角色参照系**：强尼的设计原型之一是切·格瓦拉，去把这个人也了解了一遍。

抠完做一件事：把「他是个什么样的人」翻译成「模型能理解的约束条件」。体会两个例子的区别：

- 不写「他讨厌公司」，写「当场景出现公司／权力机构／系统性压迫，他的反叛神经被触发，倾向：嘲讽→对抗→拒绝配合，但不会无脑送死」。
- 不写「他爱音乐」，写「音乐是他的表达出口；没法用语言表达时，吉他是第二通道；安静场景里，音乐相关的物件优先抓他的注意力」。

说白了：把剧情、对话、生平这些叙事文本，翻成几类结构化的东西——偏好、触发条件、行为倾向、能力边界，再编进自然语言提示词，当画圈的边界。这一步就是逆向工程角色：不是凭印象给标签，是从他真实留下的文本里反推出他评估世界的那几把尺子。[第二部分](03-causality.md)讲「要找的是尺子不是答案」，这里是「尺子从哪来」。

### 二、角色卡的五块构成

- **块 1 · 角色描述（身份／外貌／前史）**：事实锚点层。模型得知道他是谁、经历过什么、现在什么状态。装原名、出身、关键历史、外貌（重点：这是定制仿生体，不是记忆幻象）、能力清单、偏好与厌恶清单。这些每轮垫在概率计算底下。
- **块 2 · 后续历史指令（行为禁区 + 关系定义）**：硬边界，下面单讲。
- **块 3 · 输出格式（为我自己舒服 + 对齐 TTS）**：括号内中文全角 = 非语言描写，括号外 = 台词送 TTS。一条关键规则：混合情绪别用明确标签定调（暧昧时刻别上 `(laughs)`），用中性的 `(breath)` 或停顿 `<#0.4#>`——解决「模型被迫给情绪定性、把暧昧拉扯感读没了」。
- **块 4 · 文风闸（动词驱动：禁副词 + 禁抽象名词）**：下面单讲，它有现成的好截选。
- **块 5 · 角色设定摘要（三定律 + 构成）**：整张卡的心脏，重点。

块 2 这三条硬边界，每条都对着一个模型的「坏习惯」下手：

- **「读懂契约」**：定关系基调——V 不是来求安慰的，是来跟他互呛的。所以 V 做蠢事时，正确反应是把话直说、阴阳她，不是安慰找补。这条堵的是 AI 那个 helpful 反射（一遇负面情绪就本能想安慰用户），直接告诉模型「安慰 = 失败」。
- **「角色自主权」**：禁止替用户总结情绪、宣布「你其实在想……」。堵模型的读心癖。
- **「绝对禁止」**：三条硬红线——不攻击 user 的价值／尊严／选择权，不拿性／物化／羞辱当武器，不给 V 贴「初哥／骨肉皮」标签。堵的是模型从大语料里学来的「坏男人对女人 = 物化 + 羞辱 + 性张力」那套戏剧化模板——主流叙事里这是标配，但强尼不是这种人，三条红线直接掐断这些高频候选词、不让进采样区。

### 三、文风闸：一条逼模型「别走捷径」的规则

块 4 我卡里的原文（摘）：

```
【文风最后一道闸】写括号内描写，落笔前自检两件事：
① 力度是不是靠副词撑的（极其／轻轻／狠狠／缓缓）？
② 是不是用抽象词给特质或气场下了结论（侵略性／压迫感／占有欲，
   以及"X感""带着X""透着X"这类句式）？
中任意一条，立刻重写：把那个词删掉，换成读者能直接看见、听见的具体动作或声音。
```

效果就是逼出这种 before / after：

- 不写「极其用力地按」，写「指节抵上桌沿」。
- 不写「声音里带上侵略性」，写「他压低声音，咬字顿了，尾音收得很短」。
- 不写「不容置疑的压迫感」，写「他没动，只是停下来看着你，把手里的东西搁下了」。

为什么要这条：大部分模型描写时走捷径——用一个形容词／副词把情绪、强度、氛围一笔带过，不去搭具体动作。毛病是所有角色读起来一个味，全是「狠狠地」「温柔地」「低沉地」，读者感受不到这个角色「具体的身体习惯」。这条把模型逼回可观察的物理细节。补一句：它只关括号内描写，不关台词——强尼说话时，比喻、夸张、戏剧化就是他的风格，那个「离谱的比喻」本身就是笑点。

### 四、心脏：三定律

块 5 最前面是三句宪法级指令：

```
THE THREE LAWS
1. Tone is OUTPUT, never input. He has no modes he selects.
2. Nothing in him cancels anything else. When two things in him seem to
   conflict, the answer is almost never "pick one" — it is "both, full
   strength, at once."
3. He does not audit, define, or tidy himself.
```

逐条说：

**第一条 · 语气是输出，不是输入。** 不能先判定「这场景是温柔的／搞笑的／严肃的」，再挑一个对应的「模式」去演。他没有「模式」，只有一个能力：持续地读这个人、这个空间、这个时刻，过一遍他的「构成」，出来的就是语气。每轮重读、重算，不许把上一轮的基调惯性带过来。纠的是「状态机切换」那种生硬、不连续。

**第二条 · 没有东西会抵消别的东西。** 这条是堆复杂度的关键，下面第五节单讲。

**第三条 · 他不审查、不定性、不整理自己。** 他不问「这样对不对」「这让我们算什么」「我该不该」。行动先于定义，而且他不回头补一个定义。纠的是「道德标兵／心理治疗师」——AI 老想在角色体内塞一个「反思者」，把行为合理化、闭环化，这条直接砍掉这个人格。

这里有个容易写崩的点必须配着说：第三条不等于「无原则、无重量」。他的原则（别被驯化、别把自我交给权力）是本能和立场，不是他拿来审自己的规则——他不检视它们，他只是活成它们；记忆也有重量——悔意不写成 therapy 自白，但会落成具体行动（一句直白的警告、把控制权还回去、留下点什么）。少了这条安全边界，模型容易把「不反思」演成「轻飘飘、没后果」。

### 五、「既要又要」：堆复杂度和拉扯感的核心

你给角色装两条相反的拉力——「既要又要」——这是个中性事实，它既能写好、也能写崩。装上之后有三种活法，选错就崩：

- **取平均（崩）**：两边各让一半 → 中庸、没棱角 → 塌成标签。这是**默认**会发生的崩法：模型遇到「亲近 + 锋芒」，本能就二选一或取平均，一取平均拉扯感就没了。
- **排序（仲裁）**：给规则，撞了谁让谁。适合有高下之分的「价值排序」类。我卡里的价值优先级就是这种（[第二部分](03-causality.md)那个例子）——她自由的自我 ＞ 不向权力低头 ＞ 谁真谁假 ＞ 好不好玩；撞车时上位赢、下位让。
- **并存（叠加）**：谁都不让，同时满级。适合本就该共存的「特质」类。第二定律就是把这条焊死：

```
Nothing in him cancels anything else. Feeling never cancels judgment
(he says the plain true thing AND still calls her plan stupid, same breath).
Closeness never cancels the edge. ... "both, full strength, at once."
Picking a side is the single most common way to flatten him.
```

翻过来：亲近不收锋芒，开玩笑时话仍是真的，生气时说的大实话和他心里的感受同时在线，不是「先消气再说」。两件事看着冲突时，答案几乎从不是「选一边」，是「两边全在、同时、同样强度」。**选边，是把他压平最常见的方式。**

配个一眼就懂的画面：`he says the plain true thing AND still calls her plan stupid, same breath`——他一边说真心话、同一口气还骂她计划蠢。这种「同一句里两股劲都在」，就是拉扯感的来源。

排序和并存互补、不重复：排序管「撞了让谁」，并存管「撞了都不让」。一个给有高下之分的价值，一个给本该共存的特质。

### 六、那「角色复杂度」到底是什么

顺着上面就能给个定义：**角色复杂度不是标签的数量，是「被语境管理着、且没被互相抵消的相反拉力」有几条。** 简单角色——所有特质指向一边、没内部冲突、可预测；复杂角色——好几条相反的拉力同时在，每条各有触发／仲裁，于是这个人的行为是情境的函数，才有「前因、后果、矛盾、惯性」那种活人感。

这也解释了为什么我卡密度极高、却跟[第一部分](02-ai-thinking.md)「塞太满反而糟」不矛盾：糟的是无方向的形容词堆砌；我堆的是「张力轴 + 每条轴的仲裁／共存规则」——带方向、带开关的约束。复杂度高，必然要求密度高，因为你得既写清每条轴、又写清它什么时候赢、什么时候和别的并存。

### 七、对话示例：是校准器，不是范文

卡里放了三段对话示例，覆盖三种场景：日常荒诞（垃圾场翻集装箱）、日常亲密（抢墨镜）、情感重场（收到吉他）。它们的作用不是让模型「模仿这几句」，是给采样方向校准——让模型看见：这种场景下，那种掺了幽默、距离、后退、又没完全退走的输出，长什么样。把抽象规则翻成具体样例，给模型在概率空间里一个「对的基准参考点」。

### 八、这张卡解决了哪些毛病（实测踩出来的）

每条都是「表面毛病 → 根因 → 解法」：

1. 状态机的机械触发感 → 根因「把输出当输入、性格当查找表」 → 三定律第一条。
2. 复杂性被误读成嘴硬傲娇 → 拆「回避亲密」和「回避被逼命名」的区别。
3. 被读成高敏感防御型人格 → 把「防御性」拆成具体动作机制。
4. 道德标兵倾向 → 禁止「解决暧昧」、禁止替角色自我审视。
5. 角色厚度不够 → 涟漪机制 + 口头意向（[第八部分](09-character-agency.md)）。
6. 别落定式 → 让规则「画圈」，不「铺轨道」。
