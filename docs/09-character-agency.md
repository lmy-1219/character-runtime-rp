# 第八部分 · 角色自主性：一套还在建的设计

> ⚠️ **这一部分和前面不同：整套机制还没完整实测，目前只有涟漪池跑过，其余在测。** 下面是一份「在建中的设计方案」，不是交付的成品。我把设计逻辑、已完成的部分、轻测后的判断如实摆出来——当思路看，别当结论。

设计意图：现有系统（角色卡 + 世界书 + 工作流）能保证他「不走形」，但保证不了他「主动动」——他所有的移动都是对用户输入的响应，不来自他自己内部的牵引。这套就是补这块。

### 一、「自主性不足」不是一个问题，是五个

每个缺陷对应一个解法：

- ① **金鱼问题**：提过的打算一滑出上下文就消失 → 「口头意向」条目。
- ② **待机问题**：每个场景开局都在等 user 给第一拍 → DEFAULT STATE。
- ③ **通用问题**：他的主动「正确」但不「像他」，没有能力指纹和错误指纹 → fingerprint + wrong。
- ④ **无回响问题**：他独走没有后果，提了也白提 → SOLO THREADS。
- ⑤ **全靠涌现**：模型侧没有「计算他此刻要什么」这一步，主动全凭运气 → 计算步。
- （存量）**事实不发酵**：事实只当背景、不变成心理压力源 → 涟漪池 + wrecRipple（就是[第三部分](04-ripple.md)那个）。

### 二、核心架构三原则

1. **说出口的议程走事实层，没说出口的牵引走心理层。** 「强尼第三天夜里说要去找克里」是客观发生过的一次发言，合法进事实世界书——这就是「口头意向」条目，不违反「世界书只记事实」。而「那把琴让他硌得慌」是推断，只能待在涟漪池。两层永不混。
2. **生成端天马行空，记录端只认落地。** RP 里他随便提、随便讨论；但 wrecTask 的保守性（讨论 ≠ 执行）保证账本绝不因为「提过」就翻状态。他独走的后果靠「回来汇报 + 可观察的痕迹」在场内交付，你可以质疑、否决，只有活过这一场的内容才会被沉淀。
3. **驱动 = 由状态生成；表演 = 被计数器触发。** 全系统禁止任何「每 X 轮必须」。不规则性来自「牵引跟着局面走」，可靠性来自：发起设成默认态 + 计算步每拍现算 + 采样放开。

### 三、完整闭环

一个自循环：他主动发起一件事 → 你俩互动 → 事情发生（或没发生）→ 系统记事实 → 更新他的状态 → 他基于新状态再发起下一件。核心是「他先动、你回应、系统记录、状态更新、他再动」，而不是「你动、他回应」的单向。

### 四、三道隔离红线（整个系统不自我污染的命门）

这就是[第三部分](04-ripple.md)埋的伏笔——为什么涟漪池要物理隔离：

1. wrecTask 和 wrecRipple 心智相反（一个只认证据、一个专做推断），两份提示词顶部各放一句互斥声明。
2. 涟漪池绝不回流事实管线：STMB 抽取、wrecTask 导入，永远不选涟漪池条目；涟漪池物理放 Chat Lorebook，与事实世界书隔离。违反这条会产生「系统幻觉自己的尾巴」那种自我强化回路。
3. 「处置」字段手动维护，不交给任何审稿员。

为啥死磕物理隔离：涟漪池是「他心里在想什么」的推断层，不是事实层。系统要是把它当事实，就会进入「他因为想 X 所以做了 Y，而 Y 又证明他确实在想 X」的循环论证。所以涟漪池里的东西，永远不进 STMB 和 WREC 的事实管线。

### 五、天花板声明

模型没有跨时间的真实意图。这套系统的本质是：事实层持久化他说出口的议程（口头意向）+ 心理层养他没说出口的牵引（涟漪池）+ 计算步每拍现算他要什么 + 独走回响给行为以重量，四件套合力逼近「他在过自己的日子」。以我的判断，这已经贴近聊天式 LLM 在 ST 框架里的上限；再往上是写真正的 agent 框架，场内收益很小，对我来说不值当。

### 六、实测进度（老实说）

目前体感提升不会太大。因为角色卡和总结模块其实已经把自主性拉上去不少了。而且这套前期可能很不稳：可以预见，Gemini 会因为规则强调「允许」而让 Johnny 满屏乱飞，Claude 则很可能「宁愿什么都不做也不愿犯错」。还得陆续测 + 调，但因为体感提升不大、动力也不太足，这块离完整落地还有挺大距离。再强调一次：这一部分是思路和架构，不是已验证的方法，自己甄别。

### 七、落地：从「还在建」到「轻量就够了」

前面第六节写的是当时的状态——体感提升不大、动力不足、离完整落地还有挺大距离。这一节是后来的进展：它落地了。

落地之后回头看，最出乎我意料的一点是：这套东西最终是**轻量实现**的。我原本以为要给「主动性」单独搭一套重的东西（独立的意图计算员、审核、取料流水线），真做下来发现——现有架构本身就支持，绝大多数改动是往已有部件上挂东西，而不是新造机制层。计算步复用了已经在跑的召回路径；「说出口的意向」克隆了已有的世界书条目配置；「没说出口的牵引」是往滚动总结里加了个区；创伤发酵是挂在已有的关系/前史条目上。走到这一步之前，我自己都没料到这套架构有这么自洽——地基打得稳，每个部分是在互相支撑的，所以补最后这一块并不需要另起炉灶。

下面把具体的提示词工程改动按 **原文 → 修改 → 解决了什么** 列出来。提示词原文大多是英文长段，这里只摘代表性的一两句，完整版在仓库里。

#### 1. 计算步：环内、无状态、每拍现算

- **原文**：无——第五节说「计算步每拍现算他要什么」，但当时只有这个说法，没有落地的执行规程。
- **修改**：确定计算步走「环内无状态现算」这一族解法（排除了改权重、存意图、机械触发几条路）。核心是把「意图」按**是否已外化**切开：说出口/做出来的意图本就是事实，合法持久化，进议程账本（见下条 2）；没说出口的意图，它的跨拍一致性对 user 不可观测、也不可证伪，就不为它维持连续性——酝酿靠「演出来」，隐忍到爆发靠「爆发那一拍现构」。这样「怎么存意图」这个无解的两难就消失了，剩下的只是「怎么让主模型每拍可靠现算」这个提示词问题。研发路径定为 A0（常驻许可 + 两池可见）起步，不够体感再上 A1（结构化预思考、定长 `<calc>` 块），B（环外独立计算员）只作诊断示波器、输出用完即弃。现算过程包在 `<calc>…</calc>` 里，靠正则出站剥离（见下条 6），总结员也配了「遇 `<calc>` 一律无视」的规则。
- **解决了什么**：补上了第①⑤两个缺陷里模型侧「计算他此刻要什么」这一步，且没有踩「存下来的推断被反复读取再生成」那条自我污染命门。轻的地方在于：它复用现有召回路径，没有另写 agent 框架——这正是第五节说的「再往上不值当」那条线，落地时确认了不必越过。

#### 2. 已外化意图 → 议程账本

- **原文**：第②条原方案是「口头意向」条目。
- **修改**：升级为新建世界书条目「私有/Johnny-议程账本」，**克隆现有「私有/Johnny-阶段调制」条目的全部配置**（常驻、深度、位置、不回流 STMB/wrecTask），排序上紧邻阶段调制之前。条目带状态机——`进行中／搁置(N)＝被拒或被无视 N 次／悬置＝从未提起`，活跃条目上限 8 条，只由人工维护，模型不得改写。代表性一句：

  > 读法：这是 Johnny 自己心里挂着的事，不是任务清单。搁置≠放弃——"搁置"态条目仍是活的拉扯，会换姿势重现。正文中不得出现"账本/议程/条目"这类系统词——拉扯只以他的行为与话语显形。

- **解决了什么**：金鱼问题（①）。说出口的意向不再一滑出上下文就消失，而是带状态活着。轻的地方在于：它是**克隆**一个已有条目，不是新造一层。

#### 3. 未外化意图 → 滚动总结扩容 + 牵引痕 + 燃料管

- **原文**：原滚动总结是维护叙事连续性的事实便签，分区较少。
- **修改**：整体替换为六分区结构——【时间】【事实主干】【未闭环】【余震】【牵引痕】【账本草稿】。其中【牵引痕】专记「指向未来的重复性可观测行为」，行格式强制为「行为动词 + 对象 +（次数或时长）」，禁用一切心理动词与推测词：

  > 禁例："他似乎在想念克里"／"他在意水晶宫的事"——出现任何此类句式即为本区失败，改写为行为行或删除。

  再在世界书「规则/计划推进」条目里追加一段燃料管，规定主动牵引从议程账本 + 牵引痕 + 阶段调制 + 当前场景取料，且「至多一条私人牵引每回合推进，且多数回合一条都不推进」。
- **解决了什么**：没说出口的牵引有了合法的、不越过隔离红线的承载方式——它只以外化行为的形态被总结员收进牵引痕，未外化的东西不进任何持久层。轻的地方在于：这是给已经在跑的滚动总结**加了个区 + 改配额**，不是新系统。

#### 4. 行动基线 → 角色卡 THE MOTOR 常数区

- **原文**：第③条原方案是 fingerprint + wrong 指纹；行动倾向散在既有 THE MAKEUP 各块里。
- **修改**：角色卡新增一整块 THE MOTOR，定义「他如何**发起**动作」（明确写它是生成器、不是又一条要满足的规则）。要点包括：默认语法是 comment-AND-move（话搭着动作走、不替代动作）；每个独走带一个小小的自我宣称（rider，eye-roll 级、不碰她利益）；被拦截 ≠ 放弃（压力转成跟拦截的摩擦）；动手前的 half-beat 只读具体后果、不写权衡独白；偏心透镜（判决跟人走不跟事走、且自己不自知）；V 高于一切冲动但革命没被阉割；Contracts（她拿来「文书」时的固定反射）。代表性一句：

  > Default grammar is comment-AND-move: his words ride along the action, never replace it.

  另在 [Value order] 块尾追加一句，让「层级不动、每层压多重取决于台上是谁」。
- **解决了什么**：待机问题（②）与通用问题（③）。他的主动既有默认发起态、又「像他」——靠的是卡里已内置的成因尺子来校准，不另写行为映射表。轻的地方在于：这是往已有卡里写常数，用的还是「抠成因不抠映射」那套老规矩。

#### 5. 事实发酵 / 创伤 → 沉积链 + 结算块 + 余震区

- **原文**：存量缺陷「事实不发酵」原挂在涟漪池 + wrecRipple 上。
- **修改**：这里做了一个关键区分——普通事实的余震默认**向下**冷却（余震区随叙事时间粗化直至消失），但底层创伤不遵守这个规律，它不冷却、甚至可能在沉默期变重，走向由每次被触发时的具体情境决定。所以创伤不存成一个强度值、也不自动衰减，而是**每次被真实触发的当下重新结算**。落地为「沉积链」：给关系/Alt、前史/2023 两条已有条目各追加一段【链体】+【结算】，另新建「沉积/V-loss」条目。链体只存「历史上每次被触发后他做了什么」（只记行为、不记强度），结算块规定：

  > 本条被触发时：读上方事实与链体（若有），结合当下场景与他的成因，现场决定这一拍是宣泄、叠加、还是压下——方向由此刻的局面决定，不由本条预设，也不由上一次的走向决定。

  并硬性规定「只输出这一拍的外显反应，不输出认知结论」「链体是先例不是趋势线，不得默认本次烈度≥上次」「本拍算完不存档」。
- **解决了什么**：事实能发酵成心理压力源，且创伤的非线性走向（同一个创伤这次往下、下次往上）由现场结算跑出来，不被自动衰减误伤、也不被存死成定论。轻的地方在于：链体是**挂在已有世界书条目上**、复用已有的关键词触发，没有新建一套发酵引擎。

#### 6. 隔离配套 → 不回流 + 正则剥离 + 纠偏档

- **原文**：三道隔离红线（第四节）+ 各模型纠偏档（第六部分）。
- **修改**：新增的私有/议程账本、三条沉积链全部沿用「不回流 STMB/wrecTask」；`<calc>` 块靠出站正则剥离、不进聊天存档；针对两个模型的老毛病预写了症状驱动的补丁库——Claude 那边给主动性松绑（「在他自己的领域里，出手和提议就是静息态，一个只评论、只暗示、等她接话的 Johnny 和一个保姆化的 Johnny 是同种欠射」），Gemini 那边加阀（行动烈度不许跨拍单调爬升、一根针只落一次不许长成说教）。
- **解决了什么**：新增的东西全部守在隔离红线内侧，没有一处让推断回流进事实管线。轻的地方在于：模型纠偏是**并进已有的纠偏档**，几乎不新增独立规则——它擦的是两个模型各自漏色的滤镜，不是给角色写新规则。

---

这一节从头到尾，做的事情就是「把最后一块挂上去」，而不是「为最后一块另建一座楼」。回过头看，第六节那种「离完整落地还有挺大距离」的判断，落地时发现距离主要在于我当时没意识到架构已经把大半路铺好了。它落地的过程，也是我第一次真正看清这套架构有多自洽的过程。这条从「还在建、动力不足」到「发现架构本就支持、轻量就解决了」的转折是真实发生过的，我把它照实留在这里，没有改写成「我一开始就想清楚了」。
