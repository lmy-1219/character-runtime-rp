# 运行于 SillyTavern 的手搓 RP 系统：设计与底层逻辑

> 一套尽力让 AI 扮演的角色「活起来」、而且长期不走样的提示词架构。
> 实测载体是一张《赛博朋克 2077》AU 向的 Johnny Silverhand 高精度卡，迭代三个多月、实测 600+ 轮。

我对提示词的原逻辑是：抠底层、不抠映射；找模型表现的原因，不找角色标签的漏洞。也是基于这个，我认为角色设计是整个 RP 系统的原点。

后面提示词原文我大多砍了，只留典型片段配批注；想看哪段完整设计，开 issue 戳我。

这份文档记录的是一套面向 SillyTavern 长期 RP 的提示词系统：从模型的概率预测机制出发，把角色还原、角色卡、世界书、涟漪机制、模型纠偏和插件工作流串成一套可控架构。重点不在插件炫技，而在角色逻辑和提示词工程逻辑：尽量提高还原保真、高复杂度表现和长期一致性，同时把搭建后的维护成本压到可控范围内。

这是一份持续更新的个人研究记录，不是一份成品文档。仓库分成三块，各自的性质不一样：`docs/` 是已经落地、实测验证过的结论；`lab/` 是形成这些结论的过程与证据，里面有走过的弯路、还没验证的观察、以及设计完成但尚未实测、等待「毕业」的设计稿；`inspiration/` 是还没进入架构的跨领域理论线索。因此 `lab/` 与 `inspiration/` 里的内容带有不确定性——请把它们当「研究现场」来读，而不是当「成品」。
