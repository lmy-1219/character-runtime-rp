# 自主性的思考路径：怎么走到「轻量就够了」的，以及中途走岔的地方

> **状态：** 这是过程和证据，不是结论正文。结论那份在 [docs/09-character-agency.md](../docs/09-character-agency.md) 的第七节。这里忠实还原我当时的思考顺序，包括走岔的、后来被自己推翻的部分——它记的是「这套方法论怎么形成的」，不是「它有多对」。误区照实写，不美化、也不替自己判成缺陷。

自主性这块不是一次想清楚的，是隔着几个窗口、反复往下挖出来的。下面按当时的先后顺序走一遍，中间几处走岔的地方我留在原位，不挪到结论后面去圆。

## 窗口一：从「变量是不是新东西」一路问到「计算步」

起点其实跟自主性没关系。我在贴吧看到有人讲「变量 / 预设 / 数值体系」，想搞清楚它是什么、和我这套手搓的东西有没有共通逻辑。

- **第一个分叉**：变量是不是一种新机制？我判断不是——它本质是环外的键值对 + if，通过宏控制往 context 塞哪段预写好的文字，不引入任何对模型的新作用机制。定位成「存储 / 触发层工具」，和我那三支柱不在同一个抽象层。
- **第二个分叉**：变量对我的递归世界书是不是更高阶？也不是，是平级换了个触发介质。关键词触发是语义自然涌现，变量阈值触发是外部强加，踩「禁止每 X 轮必须」那条线。
- **第三个分叉，这里走过一小段岔**：变量用在涟漪层算轮数是不是浪费？我先否掉了「算轮数」——那是我早就判死的轴。然后一度接受了一个替代想法：让变量「算闭环事实」，替 LLM 扛确定性判定。但紧接着自己把它推翻了：我每 25 轮的滚动总结已经在做闭环判定，而且做得更好——LLM 现场重判 vs 变量预设枚举闭环形态，后者逮不住没预设过的闭环方式。所以变量在涟漪层对我基本没位置，我现有的「厚卡 + 强总结」已经在更高抽象层解决了同一个问题。

真正的转向在这之后：既然涟漪能被总结吸收，那能不能让总结自带时间感知？这一问把讨论从「变量」彻底拽到了自主性上。往下几步：让总结从 RP 内容推演**叙事时间**（故事内过了多久，不是轮数），判定事件悬置了多久、user 有没有回应（事实层），但绝不替角色下「所以他该生气了」这种情绪结论（那是主模型读着事实用尺子现算的）。守住这一格，涟漪作为独立机制就能优雅退役、融进总结。

顺着这条线，反应式自主性那半边基本被拿下了。剩下一个真正没解的硬核被孤立出来——**计算步**：现有系统能保证 Johnny「不走形」，但他所有移动都是对 user 输入的反应；他没法从内部牵引出一个上下文里还不存在的、朝向未来的意图。计算步就是在主模型每拍生成前，先现算「他此刻内部要什么」。

这一步我还做了一次裁剪，把 09 篇原来列的五个问题砍到只剩一个：待机问题（我是长期连续 RP、无冷启动，不存在）、指纹问题（卡厚、已内置成因尺子，已解决）、独走后果（模型有补全能力 + ST 可手改输出兜底，可接受、放后面），最后只剩计算步是硬核，而且是最前面的扳机——它不过，其余全空转。

窗口一收尾时留下的判断：计算步的疑似解是「无状态每拍现算 + 连续性外包给两个池子」，研发路径定「先 B 观测台架、后 A 固化」。但**当时这只是理论闭环，工程没闭环**，而且我自己在 09 篇写了「体感提升不大、动力不足」——这是最现实的阻塞，不是不能做，是不确定值不值得做。

## 窗口二：逼问隔离原则的边界，第三轴浮出来

这个窗口没去攻计算步，而是先做了件更底层的事：把「事实 / 推断隔离」这条总原则拿去做压力测试，看它真实边界在哪。过程里校正了两个我自己默认加上、其实不成立的约束。

- **第一次校正**：整套逻辑落下来，是不是「事实层需要全面保持纯净」？不是。真正的边界是——事实层的**内容**必须纯净（只装客观可观测、零情绪标签），但这不等于事实层的**准入范围**也得是全知客观判定。「余震只记角色当时主观经历到的」不是给事实层开主观滤网，是缩小取景范围；范围外的东西对角色而言确实没发生，「角色没经历到 X」本身就是客观事实。隔离原则没被打穿。
- **第二次校正，这条最锋利**：Johnny 式的「事实模糊了、恨反而层叠上涨」，情绪反向侵蚀事实，这个污染边界要不要重判？我拆出来的结论是：隔离原则真正保证的是「心理层不能反向伪造 / 篡改事实层的内容」（造假红线，守死），而不是「心理层的强度必须能被事实层的内容量化解释」——后面这条是我之前没意识到自己默认加上的**多余约束**，它在长期情绪沉积上不成立。撤掉它，红线不破。据此分出两条赛道：事件级涟漪 / 余震（近期、可溯源、user 能现场质疑，继续严守纯净）vs 长期情绪沉积物（允许情绪强度独立于甚至反向塑造事实记忆，不要求逐条溯源）；底层那条原始客观存档仍然不许被心理层改写。

就在这个校准过程里，一个比计算步更深的维度自己浮了出来——**第三轴**：一段旧事实，角色对它的**解读**会随认知增长、新经历累积，在很久以后被重新计算。这不是涟漪管的「事件热度」（同一件事、权重变），也不是余震管的「物理痕迹颗粒度」（同一件事、细节粗化），是**绑定在事实上的意义本身在时间轴上漂移**。当时判定它和计算步同构（无状态、只存原料、每拍现算），并给它排在计算步之后。

这个窗口中途我切进了一个亲历案例（家庭冲突的降强度版），它一次性验证了前面几条理论线，也成了第三轴落地结构的天然模板：事件当下纯事实入库、认知边界把心理留痕挡在门外 → 潜伏期事实层静默持有 → 获得新参照系后旧事实第一次被有能力解读、浮上来发酵。附带一条铁证：事后被别人扣的「心狠」「冷漠」标签，是全知视角对限知主体的心理越权裁定，反证了「全知工具不得定角色心理留痕」这条铁律的来历。

## 窗口三：事实 / 情绪分层，和一份需要去重的清单

这个窗口把事实层信息和情绪总结层信息怎么分别参与推理，落到了更实的地方：区分「情绪事件」（离散、带时间戳、可验证 → 属事实层）和「情绪状态总结」（跨时间归纳 → 要防的抽象污染）；创伤情绪按情绪事件落库、不按状态总结落库；伤害重量不预算，让它从事件的密度 / 强度 / 复现频率里在调用时涌现。

这个窗口末尾列了一份很长的「未闭环清单」（U1–U11）。**这里要标一句**：写这份清单时手头没有第二窗口的记忆，所以里面有些「未闭环」其实在前面的窗口里已经闭环了——最典型的是 U1（发酵绑叙事时间还是轮数），它在第一 / 第二窗口的推演里已经解决（总结按叙事时间单源、不按轮数）。这类「后见已闭环」的项，不当遗留卡点看。清单当时真正有价值的部分，是它把「哪些是需要我做一个决策 / 哪些是搭建复杂度 / 哪些是真的模型能力边界」区分开了，并得出一个总观察：真正「原理上走不通」的其实很少，绝大多数卡点性质是「需要我做一个明确决策」或「工程 / 人工投入」，不是根本性的不可能。

## 最后一个窗口：五条方向修正，和计算步的真正落法

这个窗口一开始就把我带进来的一个前提纠正了——我本来以为「目前的架构讨论整体是为了实现计算步」。这个前提被推翻，而且这个推翻是后面一切的地基。五条修正照实记下来（它们本身就是我之前的误判被逐条拆掉的过程）：

1. **一半产出属于反应侧、不为计算步服务。** 余震、衰减、第三轴、创伤自欺是独立成立的机制，计算步只把它们当压力源读取。分清之后，计算步的真前置集比「所有讨论过的东西」小得多。
2. **「第三轴以计算步为地基」——推翻。** 这是影响最大的一条误判修正。第三轴是反应式再解读（外部刺激把旧事实拉进上下文、用当下尺子重算意义），不需要任何生成前预计算，今天就能跑。上个窗口把「现算 / 防固化 / 无状态取料」说成计算步的基础设施，其实那三样是原则不是设施，我现有召回路径本就在执行。第三轴被从队尾提到可立即开工。
3. **「跨 RP 史相关性判定模块需自建」——降解。** 读时联结改为写时追加（总结员持一份敏化清单、隔 N 轮追加链体），那个要自建的模块直接消失；发酵浓度 = 链长 × 密度 × 近因，现读现算，「变浓却不能存浓」的张力化解。
4. **U 清单大面积坍缩。** 用「只存行为链、触发时现场结算」这一条原语去重新结算旧的未闭环清单，约三分之二当场解决。真正剩下的结构性风险只有一条：**行为棘轮**——模型偏置借合法的行为选择一点点棘轮化累积，它穿透一切存储纪律，没法在存储层根除，只能靠卡面回复力 + 审核站岗 + 反惯性锚 + 模型闸多层站岗。
5. **治理层裁决。** 杀掉「当前情绪状态全局变量」（持久化心理状态违反总原语）；「先 B 台架后 A」的顺序倒转成「A0 起步 → 失败样本 → A1 → 只在判断混乱级才上 B」。

这一段的总收获，是把「事实 / 推断隔离」这条总原则的真实边界校准清楚了：内容纯净 ≠ 取景全知，造假红线守死 ≠ 强度必须可溯源。

计算步到底怎么实现，是这个窗口的核心技术产出。先用排除法把解法空间收干净：模型层（无权重访问，关闭）→ 存储意图层（斯坦福 generative agents 那套，撞污染命门，拒绝）→ 机械触发层（违反反节奏铁律，拒绝）→ 人工共笔（兜底，非目标）→ 环内无状态现算族（唯一幸存的可行域）。上个窗口的解在这个族里、方向对，但不是族内最优——它还在「带着两难跳舞」，而最优解是让两难消失。

**两难的消解（最关键一步）**：把意图按「是否已外化」切开。

- **已外化意图**（说出口 / 做出来）：本就是事实，合法持久化。口头意向池升级成议程账本——带状态机的条目（进行中 / 搁置 N / 悬置），状态变更全由可观测事件驱动。这一个账本解决已外化意图的全部连续性问题。
- **未外化意图**：两难的核心在这里消解。关键洞察是——一个从未外化的意图，它的跨拍一致性对 user 不可观测、不可证伪，我没有义务为观众永远看不到的东西维持连续性。于是：酝酿 = 演出来（档 0 细节漏出、总结员收进牵引痕）；隐忍到爆发 = 爆发那拍现构计划（现构的出走和存了 28 拍的出走，在 user 座位上不可区分）；细节一致性不可观测即无损失；user 若追问「计划多久了」，角色以身份声称「想了很久」——说出口即成事实入账。

也就是说：心理事件（已外化）可存；心理状态（强度 / 方向 / 热度 / 优先级）必现算；稳定倾向属卡。主动侧要展开，规则就是意图要过夜、必须先变成戏。

研发路径最后定死为：A0（常驻许可 + 两池可见）起步 → 体感不够再上 A1（结构化预思考、四问定长 `<calc>`）→ B 只作诊断示波器、且输出一次性用完即弃（回流污染从「纪律问题」变成「结构上不可能」）。诊断分流也定了：块内判断对但没演出来 → 修文风；块内判断混乱 → 才上 B。

## 一点收尾

回看整条路径，它不是「学到一个新东西」，是反复确认了一件事：我在更高抽象层已经解决了同类问题，所以很多看起来要另起炉灶的能力，其实被现有部件消化掉了。中途走岔的几处（把变量当增量、把「算闭环事实」交给变量、把第三轴当成计算步的孩子、把相关性判定当成要自建的模块），共同点是我一开始都低估了现有架构的自洽程度，倾向于「再加一个部件」而不是「看看已有的接不接得住」。这些岔口我留在原位，因为方法论就是这么一步步被逼出来的——先信了「要另做」，撞回来，才认出「已经有了」。

至于最前置那个东西，到现在也还在原地：我到底有没有足够动力，为一串我自己都说「体感提升不大」的能力去搭台架。这个比任何技术问题都靠前，也不在这份路径能回答的范围里。
